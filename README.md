# Closira AI Agent — Pawfect Stay

A Python-based, four-stage AI customer support workflow built on the Groq API (llama-3.3-70b-versatile). It handles realistic inbound customer conversations for a fictional pet hotel business — answering questions strictly from an SOP, qualifying leads, detecting escalation triggers, and producing structured session summaries.

---

## The Business

**Pawfect Stay** is a fictional pet hotel and grooming salon in Austin, Texas. It was chosen as the SOP subject because pet care is emotionally driven — customers are anxious, attached, and quick to escalate. This makes the escalation and tone design more interesting to build and test than a simpler retail or booking context.

The SOP covers 13 services (boarding, daycare, grooming, puppy classes), vaccination policies, cancellation rules, check-in times, a loyalty programme, and clear escalation rules.

## Setup

### 1. Install the dependency

```bash
# Recommended: use the provided helper which creates a venv and installs deps
bash setup.sh
# Then activate the environment when you want an interactive session
source .venv/bin/activate
```

### 2. Set your Groq API key

```bash
# Mac / Linux
export GROQ_API_KEY="your-key-here"

# Windows Command Prompt
set GROQ_API_KEY=your-key-here

# Windows PowerShell
$env:GROQ_API_KEY="your-key-here"
```

Get a key at: https://console.groq.com

You can also copy `.env.example` to `.env` and load it in your shell:

```bash
cp .env.example .env
export $(grep -v '^#' .env | xargs)
```

## Running

### Option A — Live interactive chat

```bash
# If you prefer not to activate, run via the included launcher
bash run.sh
```

Starts a real-time CLI conversation with Biscuit (the AI assistant). Type your messages and press Enter. Type `exit` or `quit` to end the session. A summary is generated and logged automatically.

### Option B — Run all 5 test scenarios

```bash
# From the activated venv
python test_scenarios.py
```

Important security note: Do NOT commit API keys to the repository or share screenshots that include them. If you exposed your `GROQ_API_KEY` (for example, in an image), revoke it immediately and generate a new key at https://console.groq.com. Add any local keys to a `.env` file (which is ignored by `.gitignore`) or set them in your shell profile.

Runs every required test scenario automatically and saves a transcript for each to `test_transcripts/`. Takes around 60–90 seconds total.

## The Four Stages

| Stage | What it does |
|---|---|
| **1 — FAQ Answering** | Answers customer questions using only `sop_data.json`. If the answer isn't there, it escalates — never guesses. |
| **2 — Lead Qualification** | When booking intent is detected, asks 3 structured questions (one at a time): pet details, service interest, first visit or returning. |
| **3 — Escalation Detection** | Detects angry tone, medical questions, incidents, complaints, and out-of-SOP queries. Logs reason to `logs/escalation_log.json`. |
| **4 — Conversation Summary** | At session end, generates a structured summary: customer intent, details collected, SOP gaps, escalation status, next action. |

## SOP Data

All AI responses are grounded in `sop_data.json`. The file covers:

- Opening hours (weekdays, Saturday, Sunday)
- 13 services with exact prices (boarding, daycare, grooming tiers, puppy classes, add-ons)
- Vaccination requirements (Rabies, DHPP, Bordetella for dogs; Rabies, FVRCP for cats)
- Cancellation and check-in policies
- What to bring for boarding
- FAQ answers (meet and greet, photo updates, senior pet care, sick pet protocol)
- Loyalty programme details
- 8 explicit escalation rules

To update the SOP, edit `sop_data.json` — no code changes needed.

## Escalation Logging

Every escalation writes to `logs/escalation_log.json`:

```json
{
  "timestamp": "2025-06-01T15:34:12",
  "event": "ESCALATION",
  "trigger_msg": "My dog came home with a cut on his leg and nobody told me",
  "reason": "Customer reporting an incident during a stay — requires immediate human attention",
  "turn_number": 4
}
```

Full session data (entire conversation + summary) is saved per session under `logs/session_TIMESTAMP.json`.

## Trade-offs & Known Limitations

**What works well:**
- Escalation is output-format-based — no separate classifier, no extra API calls, no latency overhead
- SOP gaps are tracked and logged across sessions — feeds directly into SOP improvement
- Pet name extraction adds genuine personalisation with minimal code
- The SOP is fully externalised — non-technical staff can update it

**Known limitations:**
- No session persistence between runs — each `main.py` call starts fresh (production would use Redis keyed by phone number)
- Qualification stage trigger uses keyword matching, which can occasionally fire on casual mentions of price
- No streaming — responses appear all at once; streaming would feel faster in a real messaging app
- Single model, single thread — not suitable for concurrent conversations without a server layer

## Estimated API Cost

Using `llama-3.3-70b-versatile`:
- ~300–500 tokens input + ~150–300 output per turn
- A 5-turn session + summary ≈ 4,000–5,000 tokens total
- Well under $0.10 per full session at current pricing
